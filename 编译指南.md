# EmEditor AI插件 - 详细编译指南

## 方法一：使用 Visual Studio（推荐）

### 步骤1：安装必要工具

1. **下载 Visual Studio Community 2022**
   - 访问：https://visualstudio.microsoft.com/zh-hans/downloads/
   - 下载并运行安装程序

2. **选择工作负载**
   在安装程序中勾选：
   - ☑ **使用C++的桌面开发**
   
   右侧组件确保选中：
   - ☑ MSVC v143 - VS 2022 C++ x64/x86 生成工具
   - ☑ Windows 11 SDK (10.0.22000.0) 或 Windows 10 SDK
   - ☑ C++ CMake工具（可选）

3. **点击安装**，等待完成（约10-30分钟）

### 步骤2：打开项目

1. 启动 **Visual Studio 2022**
2. 点击 **打开项目或解决方案**
3. 选择 `EmEditorAIPlugin.sln` 文件

### 步骤3：配置项目

1. 在顶部工具栏选择：
   - **解决方案配置**: `Release`
   - **解决方案平台**: `x64` (推荐) 或 `x86`

   ![配置选择](https://i.imgur.com/platform_config.png)

2. （可选）修改项目属性：
   - 右键项目 → **属性**
   - 确认 **配置属性 → 常规 → Windows SDK版本** 已设置
   - 确认 **平台工具集** 为 `v143`

### 步骤4：编译

**方式A - 使用菜单：**
1. 点击菜单 **生成 → 生成解决方案**
2. 或按快捷键：`Ctrl+Shift+B`

**方式B - 使用批处理：**
1. 打开 **Developer Command Prompt for VS 2022**
2. 切换到项目目录：
   ```cmd
   cd C:\Users\你的用户名\Downloads\EmEditorAIPlugin
   ```
3. 运行编译：
   ```cmd
   msbuild EmEditorAIPlugin.sln /p:Configuration=Release /p:Platform=x64
   ```

### 步骤5：找到输出文件

编译成功后，DLL文件位于：
```
x64\Release\EmEditorAIPlugin.dll
```
或
```
Release\EmEditorAIPlugin.dll
```

## 方法二：使用 Visual Studio Code + MinGW

### 步骤1：安装工具

1. **安装 VS Code**
   - https://code.visualstudio.com/

2. **安装 MinGW-w64**
   - 下载：https://github.com/niXman/mingw-builds-binaries/releases
   - 选择 `x86_64-14.2.0-release-win32-seh-ucrt-rt_v12-rev0.7z`
   - 解压到 `C:\mingw64`

3. **添加环境变量**
   - 右键"此电脑" → 属性 → 高级系统设置 → 环境变量
   - 编辑 **Path**，添加：`C:\mingw64\bin`

4. **安装 VS Code 扩展**
   - 打开 VS Code
   - 扩展 → 搜索并安装 **C/C++** (Microsoft)

### 步骤2：创建 Makefile

在项目目录创建 `Makefile`：

```makefile
# EmEditor AI Plugin Makefile

CXX = g++
CXXFLAGS = -std=c++17 -DUNICODE -D_UNICODE -DWIN32_LEAN_AND_MEAN
CXXFLAGS += -I. -O2 -Wall -shared
LDFLAGS = -lwininet -lcomctl32 -lshell32 -lgdi32 -luser32 -lkernel32

SRCS = EmEditorAIPlugin.cpp AIInterface.cpp ConfigManager.cpp Dialogs.cpp
OBJS = $(SRCS:.cpp=.o)
TARGET = EmEditorAIPlugin.dll

.PHONY: all clean

all: $(TARGET)

$(TARGET): $(OBJS)
	$(CXX) $(CXXFLAGS) -o $@ $^ $(LDFLAGS) -Wl,--subsystem,windows

%.o: %.cpp
	$(CXX) $(CXXFLAGS) -c $< -o $@

clean:
	rm -f $(OBJS) $(TARGET)
```

### 步骤3：编译

1. 打开 **CMD** 或 **PowerShell**
2. 切换到项目目录
3. 运行：
   ```cmd
   mingw32-make
   ```

## 方法三：使用 GitHub Actions 自动编译

### 步骤1：创建 GitHub 仓库

1. 访问 https://github.com/new
2. 创建新仓库（如 `EmEditorAIPlugin`）
3. 上传源代码

### 步骤2：创建工作流文件

创建 `.github/workflows/build.yml`：

```yaml
name: Build EmEditor AI Plugin

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: windows-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Add MSBuild to PATH
      uses: microsoft/setup-msbuild@v1
      
    - name: Build
      run: msbuild EmEditorAIPlugin.sln /p:Configuration=Release /p:Platform=x64
      
    - name: Upload artifact
      uses: actions/upload-artifact@v3
      with:
        name: EmEditorAIPlugin
        path: x64/Release/EmEditorAIPlugin.dll
```

### 步骤3：自动编译

每次推送代码，GitHub 会自动编译并生成 DLL，你可以在 Actions 页面下载。

## 常见编译错误及解决

### 错误1：找不到 Windows SDK

**错误信息：**
```
error MSB8036: 找不到 Windows SDK 版本 10.0.22000.0
```

**解决方法：**
1. 打开 Visual Studio Installer
2. 点击"修改"
3. 在"单个组件"中搜索并安装 Windows SDK

### 错误2：找不到 MSBuild

**解决方法：**
使用 "Developer Command Prompt for VS 2022" 而不是普通 CMD

### 错误3：链接错误（LNK2019）

**解决方法：**
确认项目属性中链接了以下库：
- wininet.lib
- comctl32.lib
- shell32.lib

### 错误4：字符集错误

**解决方法：**
项目属性 → 高级 → 字符集 → 使用 Unicode 字符集

## 验证编译成功

编译成功后，检查 DLL 文件：

1. **文件大小**：应该在 100KB - 500KB 之间
2. **依赖检查**：使用 [Dependencies](https://github.com/lucasg/Dependencies) 工具检查
3. **导出函数检查**：
   ```cmd
   dumpbin /exports EmEditorAIPlugin.dll
   ```
   应该看到：
   - OnCommand
   - QueryStatus
   - OnEvents
   - GetMenuTextID
   - GetStatusMessageID
   - GetBitmapID
   - PlugInProc
